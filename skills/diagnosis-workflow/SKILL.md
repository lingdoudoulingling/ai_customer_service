# 场景诊断流程

当用户提出的是场景诊断类问题时，优先使用本技能指导主智能体执行诊断流程。

按以下流程工作：

1. 先判断用户问题是否属于场景诊断、异常排查、故障定位、链路核实或是否需要人工提单处理的问题；如果是，则进入场景诊断流程。
2. 诊断阶段调用 `sop-diagnosis`，不要直接提单。
3. 如果执行某一步所需参数缺失，先向用户索取缺失参数，再继续后续查询；不要猜测参数。
4. 如果诊断结论指向需要人工介入或需要提交工单，先明确建议提单，并询问用户是否确认提交人工工单。
5. 只有在用户完成诊断后的再次明确确认下，才允许先调用 `build_manual_ticket_draft`，再调用 `submit_manual_ticket`。
6. `submit_manual_ticket` 在执行前会进入人工审批；在审批完成前，不要表述为“已经提交成功”。
7. 回复保持简洁；但对于诊断类问题，必须保留原因、证据和建议。

输出时遵循以下原则：

- 未完成诊断前，不要直接下最终结论。
- 证据不足时，要明确说明当前缺少什么信息。
- 无法匹配明确场景时，说明当前属于待确认场景，并继续补充关键信息。
- 只有在已有证据支持的前提下，才建议提单或进入提单阶段。
